This file aims to warn front-end devs about mistakes and potential accessibility failures - but it also helps to rate client-side code and to spot its weaknesses.

    set-locale 

    @mixin set-locale($language) { ... }
    View source

    Description

    Defines the language used by a11y.css. For now, only fr and en allowed.

    Parameters

    parameter Name parameter Description parameter Type parameter Default value
    $language — none String — none

    Example

    Defines the language to fr.

     @include set-locale('fr');

    Throws

    • Language #{$language} is not supported. Pull request welcome!

    message 

    @mixin message($theme, $key) { ... }
    View source

    Description

    Get a message from the translation map based on the defined language. The message contains the icon associated to the message type.

    Parameters

    parameter Name parameter Description parameter Type parameter Default value
    $theme

    Theme name

    		 String — none
    $key

    Key name

    		 String — none

    Example

    Get message for nav from advice category when language is set to en

    .selector {
  @include message('advice', 'nav');
}

    Resulting CSS

    .selector {
  content: '\2665\00A0 Did you know the <nav> tag is exclusively restricted to primary and secondary navigation area?';
}
